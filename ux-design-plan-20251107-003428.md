### Modern PWA UX Research (2024-2025)

**Key Findings from Research:**

1. **Mobile-First Design:**
   - Thumb-friendly navigation (center-bottom placement)
   - Progressive disclosure (show important content first)
   - System fonts for native feel (`system-ui` font-family)
   - Bottom navigation bars (iOS/Android patterns)

2. **PWA Best Practices:**
   - Offline-first architecture (Service Worker caching)
   - Perceived speed optimization (skeleton screens, lazy loading)
   - Soft re-engagement nudges (contextual notifications)
   - Platform-native gestures (swipe, pull-to-refresh)

3. **Modern UX Patterns:**
   - Grid systems (responsive, flexible layouts)
   - Full-page headers (immersive experience)
   - Accessibility-first design (WCAG 2.1 AA)
   - Human-centric design (authentic, empathetic)

4. **Content Aggregator Patterns:**
   - Infinite scroll with virtual scrolling
   - Category-based navigation
   - Search-first approach
   - Reader view mode (distraction-free)

### User Personas (From Product Agent)

**Persona 1: Crypto Enthusiast (Sarah)**
- **Demographics:** Age 25-40, tech-savvy, crypto-native
- **Goals:** One platform for all crypto news, earn rewards for engagement
- **Pain Points:** Scattered sources, no rewards, ad-heavy platforms
- **Behaviors:** Daily crypto news consumption, active in communities, values transparency
- **UX Needs:** Fast content discovery, reward visibility, transparent ad system

**Persona 2: Content Creator (Mike)**
- **Demographics:** Age 28-45, crypto analyst, content producer
- **Goals:** Monetize content, build audience, earn from quality submissions
- **Pain Points:** Low monetization, limited reach, no direct rewards
- **Behaviors:** Submits quality content, engages with community, seeks analytics
- **UX Needs:** Easy submission flow, analytics dashboard, reward tracking

**Persona 3: Advertiser (BlockchainCo)**
- **Demographics:** Age 30-50, marketing manager, crypto project
- **Goals:** Reach crypto-native audience, transparent ad pricing, multi-chain payments
- **Pain Points:** Opaque ad pricing, high fees, limited crypto payment options
- **Behaviors:** Participates in auctions, tracks performance, multi-chain operations
- **UX Needs:** Clear auction UI, transparent pricing, performance tracking

**Persona 4: Casual Reader (Alex)**
- **Demographics:** Age 20-35, occasional crypto interest, mobile-first
- **Goals:** Quick news updates, offline reading, free access
- **Pain Points:** Too many apps, paywalls, complex interfaces
- **Behaviors:** Mobile browsing, occasional engagement, values simplicity
- **UX Needs:** Simple navigation, offline support, clean UI

**Persona 5: DAO Member (Emma)**
- **Demographics:** Age 25-40, long-term crypto user, governance participant
- **Goals:** Influence platform direction, earn governance rewards, premium features
- **Pain Points:** Limited governance, low voting power, no rewards for participation
- **Behaviors:** Active governance participation, high engagement, premium subscriber
- **UX Needs:** Clear governance UI, voting power visibility, proposal tracking

### User Flows to Document

**Flow 1: Onboarding & Authentication**
- **Trigger:** User visits site for first time
- **Steps:**
  1. Landing page ‚Üí Welcome screen
  2. Tap "Get Started" ‚Üí Social login options
  3. Select Google/Twitter/Email ‚Üí Reown AppKit modal
  4. Reown creates ERC-4337 smart account (automatic)
  5. Clerk creates user in background
  6. Email verification sent ‚Üí User clicks magic link
  7. Onboarding flow (interests, preferences) ‚Üí Skip or complete
  8. Dashboard ‚Üí Content feed
- **Pages:** Landing ‚Üí Auth ‚Üí Verification ‚Üí Onboarding ‚Üí Dashboard
- **Success Criteria:** User logged in, wallet created, email verified
- **Error Paths:**
  - Social login fails ‚Üí Retry or email fallback
  - Email verification fails ‚Üí Resend link
  - Onboarding skipped ‚Üí Default preferences

**Flow 2: Content Discovery & Reading**
- **Trigger:** User wants to read news
- **Steps:**
  1. Homepage ‚Üí Browse feed
  2. Filter by category (Tech, Crypto, Social, General)
  3. Tap article ‚Üí Reader view
  4. Read article ‚Üí Scroll down
  5. Upvote article ‚Üí Award points to submitter
  6. Bookmark article ‚Üí Save for later
  7. Share article ‚Üí Web Share API ‚Üí Earn 5 points
  8. Related articles ‚Üí Continue reading
- **Pages:** Homepage ‚Üí Reader View ‚Üí Related Articles
- **Success Criteria:** Article read, engagement tracked, points earned
- **Error Paths:**
  - Article fails to load ‚Üí Show cached version or error
  - Offline ‚Üí Show cached articles, queue actions

**Flow 3: Ad Auction Participation**
- **Trigger:** Advertiser wants to place ad
- **Steps:**
  1. Navigate to Ad Auctions ‚Üí Browse active auctions
  2. Filter by slot type (Banner, Sponsored, Promoted)
  3. Select auction ‚Üí View details (current bid, time remaining)
  4. Check USDT balance ‚Üí Top up if needed (Reown on-ramp)
  5. Place bid ‚Üí Enter amount (minimum 5% increment)
  6. Pay participation fee (1 USDT) ‚Üí Confirm transaction
  7. Bid placed ‚Üí Track status
  8. Auction ends ‚Üí Win or lose ‚Üí Refund or payment
- **Pages:** Ad Auctions ‚Üí Auction Detail ‚Üí Bid Form ‚Üí Confirmation ‚Üí Status
- **Success Criteria:** Bid placed, transaction confirmed, status tracked
- **Error Paths:**
  - Insufficient balance ‚Üí Show top-up option
  - Bid too low ‚Üí Show minimum bid requirement
  - Transaction fails ‚Üí Retry or cancel

**Flow 4: Points Earning & Conversion**
- **Trigger:** User wants to earn and convert points
- **Steps:**
  1. Submit quality article ‚Üí Receive 1,000 points (if 10+ upvotes)
  2. Receive upvotes ‚Üí Earn 10 points per upvote
  3. Daily login ‚Üí Earn 20 points (streak bonus)
  4. View points balance ‚Üí Check earning breakdown
  5. Accumulate 100,000+ points ‚Üí Conversion available
  6. Convert points ‚Üí 1,000 points = 1 USDT (minus 1% fee)
  7. Wait 7-day cooldown ‚Üí Convert again
  8. Withdraw USDT ‚Üí External wallet
- **Pages:** Points Dashboard ‚Üí Conversion Calculator ‚Üí Confirmation ‚Üí Transaction History
- **Success Criteria:** Points earned, converted, USDT withdrawn
- **Error Paths:**
  - Insufficient points ‚Üí Show minimum requirement
  - Cooldown active ‚Üí Show countdown timer
  - Conversion fails ‚Üí Retry or contact support

**Flow 5: DAO Governance Participation**
- **Trigger:** User wants to participate in governance
- **Steps:**
  1. Navigate to DAO ‚Üí Browse active proposals
  2. Filter by category (Content Moderation, Economic Policy, Features)
  3. Read proposal ‚Üí View full text, voting options
  4. Check voting power ‚Üí Meritocratic calculation displayed
  5. Vote ‚Üí Select option ‚Üí Confirm vote
  6. Track results ‚Üí View voting progress
  7. Proposal passes ‚Üí Execute (if applicable)
  8. Earn governance rewards ‚Üí Points awarded
- **Pages:** DAO ‚Üí Proposal List ‚Üí Proposal Detail ‚Üí Voting ‚Üí Results
- **Success Criteria:** Vote cast, results tracked, rewards earned
- **Error Paths:**
  - Insufficient voting power ‚Üí Show requirements
  - Proposal closed ‚Üí Show final results
  - Vote fails ‚Üí Retry or contact support

**TOTAL USER FLOWS:** 5 comprehensive flows
